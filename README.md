# Pre-Blackjack
우아한테크코스 블랙잭 미션 사전학습

## 실험1: main 기반 검증의 한계
### 해볼 것
1. 선택한 규칙을 main 메서드에서 구현
   - J, Q, K의 문장 카드 여부를 판단하고 맞을 경우 10을 반환한다.
   - 문장 카드가 아닌 경우 숫자로 변환하고 그대로 반환한다.
   
2. 테스트 케이스 3개로 실행해본다
   - 1, J, Q를 매개변수로 전달하여 출력을 확인한다.

3. 케이스를 6개로 늘려본다
   - D, 11, -1를 매개변수로 전달 해본다.
### 관찰할 것
1. 케이스가 늘어나니 어디를 수정해야 했는가?
   - 옳바르지 않은 문자의 경우 숫자로 변환할 때 예외가 발생하게 끔 수정했다.
   - 변환된 숫자의 범위를 검증해서 예외를 반환하게 하였다.
   
2. 수정하면서 놓친 것은 없는가?
   - 매개변수로 null을 입력한 경우에 대한 예외처리를 놓쳤다.

3. 이전 케이스가 여전히 동작하는지 어떻게 확인했는가?
    - 예외가 발생하는 코드를 주석 처리하고 실행시킨다.
